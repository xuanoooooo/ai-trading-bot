## 🤖 AI决策流程

### AI接收的信息
1. **K线数据** - 最近5根15分钟K线
2. **技术指标** - SMA、RSI、MACD、布林带等
3. **账户信息** - 总权益、可用余额、保证金占用率
4. **持仓状态** - 当前持仓方向、数量、盈亏
5. **杠杆机制** - 3倍杠杆说明和计算方式

### AI返回的决策
```json
{
    "signal": "BUY|SELL|HOLD",
    "reason": "分析理由",
    "stop_loss": 具体价格,
    "take_profit": 具体价格,
    "confidence": "HIGH|MEDIUM|LOW",
    "position_percent": 0到100的整数
}
```

### 仓位计算过程
1. **AI建议百分比** → 系统计算保证金
2. **保证金 × 3倍杠杆** → 实际仓位价值
3. **仓位价值 ÷ 当前价格** → 交易数量
4. **风控检查** → 确保在安全范围内

## 📊 技术指标详解

### 移动平均线 (SMA)
- **SMA5**: 5周期简单移动平均线（短期趋势）
- **SMA20**: 20周期简单移动平均线（中期趋势）
- **SMA50**: 50周期简单移动平均线（长期趋势）

### 动量指标
- **RSI**: 相对强弱指数
  - 超买: RSI > 70
  - 超卖: RSI < 30
  - 中性: 30 ≤ RSI ≤ 70

- **MACD**: 移动平均收敛发散
  - MACD线: 快线 - 慢线
  - 信号线: MACD的9周期EMA
  - 柱状图: MACD - 信号线

### 布林带 (Bollinger Bands)
- **上轨**: 中轨 + 2倍标准差
- **中轨**: 20周期SMA
- **下轨**: 中轨 - 2倍标准差
- **位置**: 价格在布林带中的百分比位置

### 成交量分析
- **成交量比率**: 当前成交量 / 20周期平均成交量
- **放量**: 比率 > 1.5
- **缩量**: 比率 < 0.5
- **正常**: 0.5 ≤ 比率 ≤ 1.5

## 🛡️ 风险管理

### 仓位管理原则
1. **最大仓位**: 80%可用资金
2. **最小仓位**: 5%可用资金
3. **强制预留**: 20%缓冲资金
4. **AI自主**: 完全由AI决定仓位大小

### 杠杆机制
- **固定杠杆**: 3倍杠杆
- **保证金计算**: 可用余额 × AI百分比
- **实际仓位**: 保证金 × 3倍
- **风险控制**: 系统自动限制最大使用率

### 止损止盈
- **AI设定**: 由AI根据技术分析设定
- **动态调整**: 根据市场情况调整
- **风险控制**: 系统会检查合理性

## 📈 交易策略

### 买入信号 (BUY)
AI可能给出买入信号的情况：
- 价格突破关键阻力位
- RSI从超卖区域反弹
- MACD金叉形成
- 成交量放大配合价格上涨
- 布林带位置从下部向上突破

### 卖出信号 (SELL)
AI可能给出卖出信号的情况：
- 价格跌破关键支撑位
- RSI进入超买区域
- MACD死叉形成
- 成交量放大配合价格下跌
- 布林带位置从上部向下突破

### 观望信号 (HOLD)
AI可能给出观望信号的情况：
- 市场方向不明确
- 技术指标相互矛盾
- 成交量萎缩
- 价格在关键位置震荡
- 等待更明确的信号

## ⚠️ 风险提示

### 市场风险
- **价格波动**: 加密货币价格波动剧烈
- **流动性风险**: 市场流动性不足时可能无法及时平仓
- **技术风险**: 网络中断、系统故障等
- **政策风险**: 监管政策变化

### 操作风险
- **API密钥泄露**: 可能导致资金损失
- **配置错误**: 错误的参数设置
- **监控不足**: 未能及时发现异常
- **资金管理**: 过度投入资金

### 风险控制建议
1. **使用子账户**: 限制最大损失
2. **小资金测试**: 先用小资金验证
3. **定期监控**: 定期检查运行状态
4. **及时止损**: 设置合理的止损点
5. **分散投资**: 不要把所有资金投入一个策略
